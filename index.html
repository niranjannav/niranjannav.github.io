GenAI & LLM Systems for Analytics

RAG-based Research Assistant for Fund Managers

Fidelity Investments 2024
Context: Fund managers at Fidelity spent 7 days per research cycle manually extracting data from SEC filings and proprietary documents. No structured pipeline for table extraction or graph generation. Research bottleneck limited fund coverage.
Constraints: Multi-format PDFs (scanned, native, tabular). Required structured output (tables + graphs). Had to integrate with existing LLM workflows. Production deployment with <5min SLA.
Decisions: Built custom PDF parsing using Pydantic for schema validation. Used vision models for table extraction instead of rule-based parsers (more robust to format variation). Implemented agentic system with Langchain for template generation. Deployed on scalable ML pipeline architecture.
Impact: Research prep time: 7 days → 5 minutes. Fund coverage: 4× increase. User satisfaction: >90%. Enabled analysts to focus on synthesis rather than data extraction.
Stack: Python, Pydantic, Vision Models, Langchain, RAG, AWS Bedrock

Text-to-SQL Platform (SQLGenie)

Fidelity Investments 2024 Patent Filed
Context: Data analysts required SQL expertise to query Snowflake data warehouse. Turnaround time for analysis: ~4 hours. Non-technical stakeholders blocked on data team availability.
Constraints: 100+ tables in Snowflake. Complex joins and business logic. Required high accuracy (>90%) to be production-viable. Sub-30s query generation latency requirement.
Decisions: Enriched Snowflake metadata with semantic descriptions via dbt. Built vector store with curated query examples for few-shot learning. Implemented syntax validation + result set comparison. Used Bedrock + Langchain for semantic parsing. Deployed via FastAPI for API access.
Impact: Query accuracy: 91% execution, 88% result match. Query time: 22s average. Data analysis turnaround: 4 hours → <1 minute. Democratized data access for non-technical users.
Stack: AWS Bedrock, Langchain, FastAPI, Snowflake, dbt, FAISS/ChromaDB
